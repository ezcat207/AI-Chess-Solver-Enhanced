The Chess Computer's Brain

What happens in that split second when AI decides where to move?

You're watching two AI agents play chess. Black just moved its queen. White pauses for half a second... then slides its knight to f3. Checkmate.

What happened in that half-second? Let's rewind and watch the thought process frame by frame.

💡
Why This Matters

Understanding how chess AI "thinks" teaches you how to evaluate tradeoffs in any AI system — search depth vs speed, accuracy vs performance, exploration vs exploitation.

Inside One Move Decision

When it's White's turn, the Minimax algorithm springs into action. Here's what happens:

1
Generate all legal moves

The engine lists every possible move White can make right now — typically 20-40 options.

2
Sort moves by promise

Captures and checks go first. This isn't random — good move ordering makes the search dramatically faster.

3
Look ahead 4 moves deep

For each candidate move, simulate what happens if we play it, Black responds, we counter, Black counters again... that's 4 ply deep.

4
Score each future position

At the end of each branch, evaluate who's winning. Material, position, king safety — all factors get weighted and summed.

5
Pick the move with the best worst-case

Minimax assumes Black plays perfectly. So we pick the move where even if Black responds optimally, we're still in the best position.

What This Looks Like in Code

Here's the actual function that makes the decision. Left side: real code from the project. Right side: what it means in plain English.

CODE 
let moves = engine.generateLegalMoves();
moves = this.orderMoves(engine, moves);

let bestMove = moves[0], bestScore = -Infinity;
for (const m of moves) {
  const u = engine.makeMove(m);
  const s = -this.minimax(engine, depth - 1, -Infinity, Infinity, false);
  engine.unmakeMove(u);
  if (s > bestScore) { bestScore = s; bestMove = m; }
}
PLAIN ENGLISH

Get the full list of moves White can legally make right now.

Sort them so promising moves (captures, checks) get searched first.

Start with the first move as our default choice, and give it the worst possible score.

Loop through every candidate move one by one.

Temporarily make the move on the board (we'll undo it in a moment).

Look ahead and ask: if we play this, what's the best Black can respond with? The minus sign flips the perspective.

Undo the move — the board is back to its original state.

If this move scores better than what we've seen so far, remember it as the new best.

After testing all moves, return the one with the highest score.

🎯
The Minus Sign Trick

Notice -this.minimax(...)? That's flipping the score because we're switching whose turn it is. What's good for Black is bad for White, so we multiply by -1. This is how Minimax handles turn-taking without writing separate "maximize" and "minimize" functions.
